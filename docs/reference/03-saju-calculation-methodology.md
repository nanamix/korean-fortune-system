# 03. 사주 계산 방법론 (Saju Calculation Methodology)

> 사주팔자 4주(년·월·일·시)와 십신·지장간·12운성·대운을 **정통 명리 기준**으로 산출하는 알고리즘 명세.
> 관련: [아키텍처](02-architecture.md) · [데이터 모델](04-data-model.md) · [API](05-api-reference.md)

## 요약

2026-07-11 리팩터링으로 계산 엔진을 **검증된 사주 라이브러리 `lunar-java`(cn.6tail:lunar)** 위임으로 재작성했다. 그 이전 코드는 **4주 8글자가 사실상 전부 오답**이었다(일주 기준일 상수 오류, 월주가 절기를 무시, 오호둔·오서둔 미적용, 십신/12운성/대운 기능 부재, 음력 변환은 `plusDays(30)` 근사). 현재 엔진은 lunar-java 로 4주·십신·지장간·12운성·대운·절기를 산출하며, 전문 만세력 서비스(척척사주)의 결과와 일치함을 검증했다.

핵심 클래스:

| 클래스 | 책임 |
|---|---|
| `GanjiCalculatorService` | lunar-java `EightChar` 위임 + 결과를 한글 간지로 매핑 |
| `lunar-java` (`com.nlf.calendar`) | 4주·십신·지장간·12운성·대운·절기 표준 계산 엔진 |
| `LunarSolarConverter` | 한국 음력 입력 → 양력 변환 (Time4J `KoreanCalendar`, KASI 정합) |

> 아래 §1~§8은 lunar-java 가 내부적으로 수행하는 명리 규칙을 설명한 것이며, 애플리케이션 코드가 직접 구현하지는 않는다(십신·대운 12운성 파생 헬퍼 제외). 계산 흐름: `SajuRequest` → (음력이면 Time4J 로 양력화) → 경도보정 −30분 → `new Solar(...)` → `getLunar().getEightChar()` → 한글 매핑.

## 1. 연주(年柱) — 입춘 기준

사주의 연(年)은 양력 1월 1일도 음력 설날도 아닌 **입춘(立春) 절입 시각**에 바뀐다.

```
입춘 절입시각 = 태양황경 315° 도달 시점   // lunar-java 가 내부 산출
사주연 = (출생시각 < 입춘) ? 출생연도 - 1 : 출생연도
연간 index = (사주연 - 1984) mod 10   // 1984 = 甲子년
연지 index = (사주연 - 1984) mod 12
```

1984년 갑자년을 앵커로 삼는다. 1월~입춘 이전 출생자는 전년도 간지가 된다.

## 2. 월주(月柱) — 절기 + 오호둔(五虎遁)

**월지(月支)는 24절기 중 12절(節)의 절입을 경계**로 정한다(중기 아님). 태양 황경으로 직접 판정한다.

| 절(節) | 황경 | 월지 |
|---|---|---|
| 입춘 | 315° | 寅(인) |
| 경칩 | 345° | 卯(묘) |
| 청명 | 15° | 辰(진) |
| … | … | … |
| 소한 | 285° | 丑(축) |

```
월지 = ((태양황경 - 315 + 360) mod 360) / 30 → 0=寅 …   // lunar-java 가 내부 산출
```

**월간(月干)은 오호둔(월두법)** 으로 연간에서 유도한다.

| 연간 | 寅월 천간 |
|---|---|
| 甲·己 | 丙 |
| 乙·庚 | 戊 |
| 丙·辛 | 庚 |
| 丁·壬 | 壬 |
| 戊·癸 | 甲 |

```
寅월 천간 = ((연간 mod 5) × 2 + 2) mod 10
월간 index = (寅월 천간 + (월지 - 寅)) mod 10    // 寅부터 월지까지 순행
```

## 3. 일주(日柱) — 율리우스일 60갑자

일진(日辰)은 절기·달과 무관하게 **60일 주기로 끊김 없이 순환**한다. 그레고리력 → 율리우스일수(JDN)로 계산한다.

```
JDN = day + (153·m + 2)/5 + 365·y + y/4 − y/100 + y/400 − 32045
  (a = (14−month)/12, y = year + 4800 − a, m = month + 12a − 3)

일진 index(0=甲子) = (JDN + 49) mod 60
일간 = index mod 10,  일지 = index mod 12
```

앵커 상수 `+49`는 1981-03-20(JDN=2,444,684) → index 33 = 丁酉 로 검증했다.

**자시(子時) 조기전환**: 진태양시 23:00 이후 출생은 일주가 다음날로 넘어간다(정통파). `solarTime.getHour() >= 23`이면 `effectiveDate + 1`. (야자시/조자시 분리 유파는 옵션 미구현 — [한계](#한계와-확장) 참조.)

## 4. 시주(時柱) — 오서둔(五鼠遁) + 진태양시

12시진은 2시간 단위이며 **자시 = 23:00~01:00**이다. 시각은 먼저 경도 보정한다.

```
진태양시 = 출생시각 − 30분   // 한국(동경135° 표준시) vs 실경도 약 127.5° → 약 −30분
시지 index = ((진태양시 시(hour) + 1) / 2) mod 12   // 자시=23~01
```

**시간(時干)은 오서둔(시두법)** 으로 일간에서 유도한다.

| 일간 | 子시 천간 |
|---|---|
| 甲·己 | 甲 | 乙·庚 | 丙 | 丙·辛 | 戊 | 丁·壬 | 庚 | 戊·癸 | 壬 |

```
子시 천간 = (일간 mod 5) × 2 mod 10
시간 index = (子시 천간 + 시지 index) mod 10
```

## 5. 십신(十神)

일간 대비 대상 천간의 오행 생극 + 음양으로 판정한다.

```
오행(stem) = stemIndex / 2   // 0목 1화 2토 3금 4수
음양: stemIndex 짝수=양, 홀수=음
rel = (대상오행 − 일간오행) mod 5
  0 → 비견/겁재   (동일 오행)
  1 → 식신/상관   (일간이 生)
  2 → 편재/정재   (일간이 剋)
  3 → 편관/정관   (일간을 剋)
  4 → 편인/정인   (일간을 生)
음양이 같으면 앞(비견·식신·편재·편관·편인), 다르면 뒤(겁재·상관·정재·정관·정인)
```

지지의 십신은 **지장간 정기(본기)** 로 판정한다. 예: 丁 일간에서 辛 = 편재(火剋金, 음음 동일).

## 6. 지장간(支藏干)

지지별 숨은 천간(표시 순서 = 여기/중기/정기, 마지막이 본기):

| 지지 | 지장간 | | 지지 | 지장간 |
|---|---|---|---|---|
| 子 | 壬癸 | | 午 | 丙己丁 |
| 丑 | 癸辛己 | | 未 | 丁乙己 |
| 寅 | 戊丙甲 | | 申 | 戊壬庚 |
| 卯 | 甲乙 | | 酉 | 庚辛 |
| 辰 | 乙癸戊 | | 戌 | 辛丁戊 |
| 巳 | 戊庚丙 | | 亥 | 戊甲壬 |

## 7. 12운성(十二運星)

일간의 지지에 대한 포태법(胞胎法). 장생지에서 시작해 **양간 순행 / 음간 역행**.

```
장생지(일간별): 甲亥 乙午 丙寅 丁酉 戊寅 己酉 庚巳 辛子 壬申 癸卯
양간: stageIdx = (지지 − 장생지) mod 12
음간: stageIdx = (장생지 − 지지) mod 12
순서: 장생 목욕 관대 건록 제왕 쇠 병 사 묘 절 태 양
```

예: 丁 일간, 酉 → 장생. 卯 → 병. 丑 → 묘.

## 8. 대운(大運)

```
방향: (양년간 & 남) 또는 (음년간 & 여) → 순행 / 그 외 → 역행
대운수: 인접 절(節)까지 일수 ÷ 3   (나머지 1 버림, 2 올림, 최소 1)
       순행이면 다음 절까지, 역행이면 직전 절까지
대운 간지: 월주 60갑자 index 에서 순행 +1 / 역행 −1 씩 10주
```

## 절기·절입 정밀도

절기 절입 시각은 lunar-java 가 천문 계산으로 산출한다(별도 하드코딩 테이블 없음). 라이브러리는 널리 사용·검증된 구현이며, 4주·대운 경계를 자체 처리한다.

## 음양력 변환

한국 음력 입력은 `LunarSolarConverter` → Time4J `KoreanCalendar`(KASI 정합)로 양력 변환한 뒤 lunar-java 에 투입한다. 중국력 기준인 lunar-java 대신 한국력을 신뢰원으로 쓰기 위함이다. 윤달 지원, 소월(29일)·존재하지 않는 음력 30/31일 입력은 그 달 말일로 클램프한다.

## 한계와 확장

- **자시 처리**: lunar-java 기본 유파(sect) 사용. 야자시/조자시 분리 옵션은 `EightChar.setSect(1|2)`로 확장 가능하나 현재 미노출.
- **용신(用神)·신강신약 판정 없음**: 오행 개수 집계까지만.
- **경도 보정 고정(−30분)**: 출생지별 경도 입력·균시차 보정 미구현(한국 출생 가정).
- **한국 서머타임(1948~1988) 미반영**: 해당 구간 출생자 시각 −1h 보정 로직은 추후 필요.
- **중국력/한국력 경계 차이**: 사주 4주는 lunar-java(중국력 기준 절기), 음력 날짜 입력은 Time4J(한국력)로 처리한다. 두 역법의 삭·절기 경계는 드물게 하루 차가 날 수 있다.
